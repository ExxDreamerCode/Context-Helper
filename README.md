# 🧰 ContextHelper

</div>

## 🔄 Общая архитектура

ContextHelper — это **модульная система**, которая интегрируется в контекстное меню Windows. Всё работает по следующей схеме:

```
Правый клик по файлу → Контекстное меню → ContextHelper → Выбор модуля → Обработка
```

## 📁 Компоненты системы

| Компонент | Файл | Что делает |
|-----------|------|------------|
| **Реестр Windows** | (системный) | Добавляет пункт "ContextHelper" в контекстное меню |
| **Загрузчик** | `launcher.bat` + `launcher.py` | Принимает вызов от системы, накапливает файлы, запускает модуль |
| **Менеджер реестра** | `registry_manager.py` | Устанавливает/удаляет записи в реестре |
| **GUI менеджер** | `modules_manager.py` | Графический интерфейс для включения/отключения модулей |
| **Модули** | `modules/*.py` | Конкретные обработчики (конвертация, водяной знак и т.д.) |
| **Конфиг** | `modules_config.json` | Хранит список включённых модулей |

## ⚙️ Пошаговый сценарий работы

### Шаг 1: Запуск менеджера и выбор модулей

Пользователь запускает `modules_manager.py`:

```
1. Программа сканирует папку modules/
2. Читает из каждого .py файла переменные MODULE_ID и NAME
3. Показывает список модулей с чекбоксами
4. Пользователь отмечает нужные → сохраняет → конфиг обновляется
```

### Шаг 2: Установка в контекстное меню

Пользователь запускает `registry_manager.py` **от имени администратора**:

```
1. Проверяет права администратора
2. Читает modules_config.json (какие модули включены)
3. Очищает старые записи в реестре (если были)
4. Создаёт новые ключи в HKEY_CLASSES_ROOT:
   
   HKEY_CLASSES_ROOT
   └── *\shell\MyTools\          ← для всех файлов
       ├── (Default) = "ContextHelper"
       ├── ExtendedSubCommandsKey = "*\shell\MyTools\menu"
       └── menu\shell\
           ├── to_png\
           │   ├── (Default) = "Конвертировать в PNG"
           │   └── command\ (Default) = "launcher.bat to_png %1"
           ├── watermark\
           │   └── ... (аналогично)
           └── remove_bg\
               └── ...
   
   Аналогично для AllFileSystemObjects\shell\MyTools\ (папки + файлы)
```

### Шаг 3: Пользователь щёлкает по файлу

```
1. Пользователь: правый клик по image.jpg
2. Windows показывает контекстное меню
3. Появляется подменю "ContextHelper"
4. В нём список активных модулей: 
   - Конвертировать в PNG
   - Удалить фон
   - Добавить водяной знак
   - ...
```

### Шаг 4: Выбор модуля

```
1. Пользователь выбирает, например, "Удалить фон"
2. Windows запускает: launcher.bat remove_bg "C:\path\to\image.jpg"
3. launcher.bat вызывает: python launcher.py remove_bg "C:\path\to\image.jpg"
```

### Шаг 5: Работа загрузчика (launcher.py) — ВАЖНО!

`launcher.py` решает проблему массовой обработки:

```python
# Проблема: при выборе нескольких файлов Windows вызывает программу 
# для КАЖДОГО файла отдельно (10 файлов = 10 запусков)

# Решение: launcher.py накапливает файлы

1. Создаёт lock-файл (PID файл) — кто первый, тот "главный"
2. Если lock нет → процесс становится главным:
   - Создаёт временную папку для очереди
   - Ждёт 0.5-1 секунду (подбирает другие процессы)
   - Собирает все файлы из очереди
   - Удаляет дубликаты
   - Запускает модуль ОДИН раз со всеми файлами

3. Если lock уже существует → процесс дополнительный:
   - Просто добавляет свой файл в очередь
   - Завершается (не запускает модуль)

Результат: при выборе 10 файлов модуль запустится 1 раз со всеми 10 файлами
```

### Шаг 6: Запуск модуля

`launcher.py` вызывает `module.process(files)`:

```python
# Пример: modules/remove_bg.py

def process(files):
    # files = ["1.jpg", "2.jpg", "3.jpg"] — все выбранные файлы
    
    for filepath in files:
        # 1. Читаем изображение
        # 2. Удаляем фон через rembg
        # 3. Сохраняем как *_no_bg.png
        # 4. Выводим прогресс в консоль
```

### Шаг 7: Вывод результата

```
В консоли (если видна) появляется:

  → image1... ✓ сохранён как image1_no_bg.png
  → image2... ✓ сохранён как image2_no_bg.png
  → image3... ✗ Ошибка: файл повреждён

Итог: 2 обработано, 0 пропущено, 1 ошибок
```

Некоторые модули (`duplicate_finder`) открывают отдельное GUI-окно.

## 🧩 Как работают отдельные модули

### to_png.py (Конвертировать в PNG)

```python
1. Открывает изображение через PIL.Image.open()
2. Сохраняет в том же месте с расширением .png
3. Исходный файл не удаляется
```

### watermark.py (Добавить водяной знак)

```python
1. Открывает изображение в режиме RGBA
2. Создаёт прозрачный слой-оверлей
3. Рисует текст заданным шрифтом и размером
4. Устанавливает прозрачность (OPACITY)
5. Накладывает оверлей на изображение (alpha_composite)
6. Сохраняет поверх исходного файла (или рядом)
```

### remove_bg.py (Удалить фон)

```python
1. Читает изображение в бинарном режиме
2. Передаёт в rembg.remove()
3. Получает изображение с прозрачным фоном (PNG)
4. Сохраняет как *_no_bg.png
```

### duplicate_finder.py (Найти дубликаты)

```python
1. Обходит все выбранные папки рекурсивно
2. Группирует файлы по размеру
3. Для групп с одинаковым размером вычисляет MD5 хеш
4. Находит точные дубликаты (хеш совпадает)
5. Открывает tkinter окно со списком групп
6. Пользователь отмечает и удаляет лишние копии
```

### metadata_extractor.py (Извлечь метаданные)

```python
1. Обходит все выбранные папки и файлы
2. Для каждого файла определяет тип по расширению:
   - Изображения → PIL + EXIF
   - Аудио → mutagen
   - PDF → PyPDF2
   - DOCX → python-docx
3. Собирает метаданные в словарь
4. Спрашивает пользователя: JSON или CSV?
5. Экспортирует в файл с временной меткой
```

### temp_cleaner.py (Очистить папку Temp)

```python
1. Находит все Temp-папки на компьютере:
   - Личная папка пользователя (%USERPROFILE%\AppData\Local\Temp)
   - Системная папка (C:\Windows\Temp)
   - Temp-папки всех пользователей (через реестр ProfileList)

2. Сканирует каждую папку и показывает размер

3. При нажатии "Очистить":
   - Удаляет файлы старше 24 часов
   - Очищает пустые папки
   - Показывает сколько места освобождено

4. Вывод через tkinter окно:
   - Список всех найденных Temp-папок
   - Размер каждой папки до/после очистки
   - Количество удалённых файлов
   - Сколько места освобождено

5. Защита от повторных запусков:
   - Использует lock-файл
   - При выборе нескольких файлов открывается только одно окно
   - Предыдущие процессы автоматически завершаются
```

## 🔧 Управление конфигурацией

`modules_config.json`:

```json
{
  "_comment": "true = включён, false = отключён",
  "enabled": {
    "to_png": true,
    "watermark": false,
    "remove_bg": true,
    "metadata_extractor": true,
    "duplicate_finder": true,
    "temp_cleaner": true
  }
}
```

Как это используется:

```python
# registry_manager.py
enabled = config["enabled"]

for module in all_modules:
    if enabled.get(module.id):  # true
        # Добавляем в контекстное меню
    else:  # false
        # Пропускаем (не показываем в меню)
```

## 🧼 Очистка реестра

`clean_registry.py` при запуске:

```
1. Проверяет права администратора
2. Перебирает все возможные пути в реестре:
   - *\shell\MyTools
   - *\shellex\ContextMenuHandlers
   - Directory\shell\MyTools
   - и т.д.
3. Удаляет ключи: MyTools, ContextHelper, WatermarkAction
4. Предлагает перезапустить Проводник
```

## 📊 Полный цикл работы программы

```
Установка:
python modules_manager.py → выбрать модули → сохранить
python registry_manager.py (админ) → запись в реестр
taskkill /f /im explorer.exe && start explorer.exe → перезапуск Проводника

Использование:
ПКМ по файлу → ContextHelper → Выбрать модуль
    ↓
launcher.bat → launcher.py (накопление файлов)
    ↓
modules/[модуль].py → process(files)
    ↓
Результат (новый файл / консольный вывод / GUI окно)

Удаление:
python clean_registry.py (админ) → удаление из реестра
```

## 🎯 Ключевые особенности работы

| Особенность | Как реализовано |
|-------------|-----------------|
| **Модульность** | Каждый модуль — отдельный .py файл с переменными MODULE_ID и NAME |
| **Пакетная обработка** | launcher.py накапливает файлы через lock-файл + очередь |
| **Гибкое включение** | modules_config.json + modules_manager.py |
| **Поддержка папок** | Модуль сам решает, как обрабатывать папки (os.walk) |
| **Обработка ошибок** | try/except в каждом модуле, ошибки в stderr |
| **Администратор** | registry_manager.py проверяет права через создание тестового ключа |
